= Fixing the Problem

Now that we've identified the issues with our web server, it's time to fix them. We discovered two problems in the previous module:

1. **Incorrect ownership** - `/var/www/html` is owned by `1002:1002` instead of `root:root`
2. **Incorrect SELinux context** - The directory has `default_t` instead of the proper httpd context

Let's fix these issues methodically, testing after each change.

== Step 1: Fix the File Ownership

The first issue we'll tackle is the ownership. The directory and its contents should be owned by `root:root`. We'll use the `chown` command to fix this.

[source,bash,role="execute",subs=attributes+]
----
sudo chown -R root:root /var/www/html
----

The `-R` flag makes the command recursive, so it will change ownership of the directory and all files within it.

=== Verify the Ownership Change

Let's verify that the ownership is now correct:

[source,bash,role="execute",subs=attributes+]
----
sudo ls -l /var/www/
----

You should now see:

[source,subs=attributes+]
----
[rhel@rhel ~]$ sudo ls -l /var/www/
total 0
drwxr-xr-x. 2 root root  6 Dec 10 00:00 cgi-bin
drwxr-xr-x. 3 root root 38 Mar 17 19:29 git
drwxr-xr-x. 2 root root 24 Dec 10 00:00 html
[rhel@rhel ~]$
----

Good! The ownership is now `root:root`. But if you check the `Web Site` tab, you'll see it's still not working. That's because we still need to fix the SELinux context.

== Step 2: Fix the SELinux Context

SELinux contexts define what processes can access what files. The `httpd` process needs files in `/var/www/html` to have the correct SELinux type.

We could manually set the context with `chcon`, but there's a better way: `restorecon`. This command restores the default SELinux contexts based on the system's policy. This is the preferred method because it ensures consistency with the system's SELinux policy.

[source,bash,role="execute",subs=attributes+]
----
sudo restorecon -vFR /var/www/
----

Let's break down these flags:

* `-v` - Verbose, shows what it's changing
* `-F` - Force, resets contexts even if they appear correct
* `-R` - Recursive, processes directories and their contents

=== Understanding the Output

The `restorecon` command will show you what it changed:

[source,subs=attributes+]
----
[rhel@rhel ~]$ sudo restorecon -vFR /var/www/
Relabeled /var/www/html from system_u:object_r:default_t:s0 to system_u:object_r:httpd_sys_content_t:s0
Relabeled /var/www/html/index.html from system_u:object_r:default_t:s0 to system_u:object_r:httpd_sys_content_t:s0
[rhel@rhel ~]$
----

Notice that both the directory and the `index.html` file were changed from `default_t` to `httpd_sys_content_t`. This is the correct SELinux type for httpd content.

=== Verify the SELinux Context

Let's verify the context is now correct:

[source,bash,role="execute",subs=attributes+]
----
sudo ls -lZ /var/www/
----

You should see:

[source,subs=attributes+]
----
[rhel@rhel ~]$ sudo ls -lZ /var/www/
total 0
drwxr-xr-x. 2 root root system_u:object_r:httpd_sys_script_exec_t:s0  6 Dec 10 00:00 cgi-bin
drwxr-xr-x. 3 root root system_u:object_r:git_content_t:s0           38 Mar 17 19:29 git
drwxr-xr-x. 2 root root system_u:object_r:httpd_sys_content_t:s0     24 Dec 10 00:00 html
[rhel@rhel ~]$
----

Perfect! The `html` directory now has the correct `httpd_sys_content_t` context.

== Step 3: Test the Web Site

Now that we've fixed both issues, let's test the web site. Click on the `Web Site` tab at the top of the terminal.

You should now see the Super-Business website with the message "This is our super-businessy important web site" instead of the default Apache test page!

[TIP]
====
If you still see the default Apache page, try refreshing your browser. Sometimes browsers cache the error page.
====

== What We Learned

In this lab, you learned how to:

1. **Systematically diagnose** a web server issue by:
   * Checking configuration files
   * Verifying file locations
   * Examining ownership and permissions
   * Checking SELinux contexts

2. **Fix common issues** with Linux web servers:
   * Using `chown` to correct file ownership
   * Using `restorecon` to restore proper SELinux contexts

3. **Work methodically** - We gathered information first, identified the problem, and then made targeted fixes rather than blindly changing things.

[IMPORTANT]
====
Always investigate before you fix! Rushing to make changes without understanding the problem can make things worse. The time spent gathering information is never wasted.
====
